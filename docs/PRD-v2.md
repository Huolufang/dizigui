# 弟子规学堂 PRD v2.0

> 状态：已确认基线  
> 生效日期：2026-07-17  
> 最后更新：2026-07-17  
> 主要读者：产品所有者  
> 文档用途：作为后续课程重编、界面调整、数据改造和验收的唯一需求依据

## 1. 产品定义

### 1.1 背景与问题

现有 v1.1.0 已经验证了课程、讲稿、背诵、复习、反思和学习记录等基本功能形态，但其需求规划存在以下问题：

- 将儿童、家长和成年人等不同使用者混在同一套语言和流程中；
- 以“21天”和固定步骤作为结构，课程划分与蔡老师40讲缺少精确对应；
- 把完成天数、文字长度和相似度分数当作学习成效；
- 完整讲稿、基础释义和产品编写内容的来源边界不够清楚；
- “AI助手”、提醒和部分学习指标并没有对应真实能力；
- 学习记录只保存在本机，却缺少可靠备份恢复。

v2.0 不以修补旧页面为目标，而是重新定义产品结构。v1.1.0 只作为功能原型和内容输入，允许调整页面、流程和数据模型，不要求保持旧流程兼容，也不迁移旧学习状态。

本产品要解决的核心问题是：产品所有者国学基础较弱，希望系统学习《弟子规》，但缺少一条能够把经典原文、基础释义和蔡老师《细讲弟子规》40讲建立清晰对应的学习路径。

### 1.2 目标用户与使用场景

产品唯一目标用户是产品所有者本人：具备阅读能力、对传统文化感兴趣、希望系统学习《弟子规》的成年人。

典型使用场景包括：

- 按建议顺序完成一课约20分钟的系统学习；
- 需要时自由查阅任意课程、原文或完整讲次；
- 通过隐藏提示进行口头背诵，并自行判断熟练程度；
- 处理今天到期的复习内容；
- 搜索原句、注释或讲稿关键词并定位来源；
- 记录自己对讲稿和原文的理解；
- 在清除数据或更换设备前手动备份学习记录。

产品不要求每日一课，不限定完成天数，也不通过打卡、排名或强制解锁推动使用。

### 1.3 产品目标与学习成果

**一句话定位：**一款仅供个人使用、完全离线的《弟子规》系统学习工具。

产品目标是提供一条来源清晰、内容完整、可自由查阅且能够持续复习的个人学习路径。

学习成果按以下顺序组织：

1. **理解原文**：能够结合字词注释和基础译文学习课程覆盖的全部原文；
2. **学习讲解**：能够找到并阅读蔡老师对相关原文的直接讲解；
3. **完成背诵**：通过反复回忆逐步熟悉核心原文；
4. **形成反思**：保留自己从讲稿和原文中得到的理解与生活思考。

产品成功不以连续使用天数或能力分数衡量，而以以下事实判断：

- 全部经审核课程能够完整学习；
- 任意解释和讲稿段落可以追溯来源；
- 用户能够持续处理到期复习；
- 学习位置、评级和反思不会因正常退出而丢失；
- 个人学习数据可以备份并可靠恢复。

### 1.4 产品原则

1. **来源先于包装**：所有解释和讲解必须标明来源，产品不自行扩写现代观点。
2. **学习逻辑先于材料顺序**：课程按原文和完整语义组织，40讲作为可映射、可完整查阅的讲稿资料库。
3. **完整理解先于固定数量**：课程总数以内容划分结果为准，不为凑21课破坏语义。
4. **引导先于强制**：提供建议顺序，但全部内容始终可以自由查阅。
5. **行动记录先于虚假评分**：完成状态来自明确学习动作；背诵结果来自自我评级，不伪装成客观能力测量。
6. **长文阅读优先**：字号、行距、定位和阅读恢复应服务于完整讲稿阅读。
7. **可追溯优先**：课程、原文、讲次和段落使用稳定关系，任何内容都能返回来源。
8. **数据安全优先**：自动保存、备份、回滚和异常隔离优先于增加外围功能。
9. **离线与克制**：只保留个人学习真正需要的能力，不建设账号、云服务或联网依赖。

### 1.5 非目标

本产品不服务以下目标：

- 面向公众发布或获取用户；
- 儿童启蒙、亲子陪学或家长监督；
- 学校教学、班级管理或教师布置任务；
- 成人国学通识平台或其他经典扩展；
- 商业化、付费课程或内容分发；
- 对蔡老师讲稿进行自动总结、观点改写或现代化再解释；
- 用自动分数证明理解、背诵或道德实践能力；
- 用游戏化、排名、打卡或提醒提高留存；
- 取代纸书、正式出版注释本或原始讲稿来源。

## 2. 内容与课程体系

### 2.1 内容来源与定位

产品使用三类内容，三者必须独立保存、明确标注，不得合并成无法追溯来源的统一正文。

1. **《弟子规》原文**
   - 是课程划分、学习和背诵的固定对象。
   - 以《弟子规》题解·注释·译文（学习整理版）中的【原文】区块作为规范原文候选，以 v1.1.0 `data/course.js` 中的原文单元作为对照来源。两者存在字词、篇章名或标点差异时，必须逐项列入异文清单，由产品所有者选择采用文本并记录理由。
   - 内容编排开始前，根据已裁决的异文清单生成受控规范原文文件，由产品所有者逐句确认，登记版本与文件校验值；此后该文件是完整性验收的唯一底本。
   - 每个原文语义单元必须具有稳定 ID，供课程、搜索、复习和笔记引用。
2. **蔡礼旭老师《细讲弟子规》40讲**
   - 是课程解释和生活启示的主要来源。
   - 当前输入为 v1.1.0 数据文件中的40讲全文；现有文件尚未记录原始出版物、网页或整理版本信息，产品必须保留这一来源状态，不得虚构版本信息。
   - 保存一份不可修改的原始正文。供阅读的展示版本可以修正能够明确判断的错字、标点和分段问题，但必须保留修改记录，不得进行摘要式改写。
3. **《弟子规》题解·注释·译文（学习整理版）**
   - 是讲稿未覆盖部分的字词注释和基础译文底稿。
   - 必须保留文件自带的来源说明，并明确其为公开学习资料整理版，不宣传为权威定本或蔡老师观点。
   - 保存底稿原件；课程可以按原文语义单元引用对应片段，但不得静默改写注释或译文。

系统必须为每类来源建立来源登记信息，至少包括来源 ID、名称、内容类型、输入文件、版本状态、用途和备注。所有进入课程的数据都必须能够反向定位到来源。

### 2.2 内容解释优先级

内容解释遵循以下优先级：

1. 《弟子规》原文是固定学习对象，任何讲解均不得改变原文字句。
2. 蔡老师讲稿是主要解释与生活启示来源。
3. 公开学习底稿用于补充讲稿未覆盖的字词注释和基础译文。
4. 课程标题、导语及通用反思问题属于导航性编辑内容，不得新增解释性观点，也不得冒充讲稿观点。

当讲稿与公开学习底稿对同一原文存在不同理解时：

- 主课程采用蔡老师讲稿中的解释；
- 数据中必须同时保存底稿原文、讲稿原文和差异说明；
- 只有差异会影响学习者理解时，课程页面才显示简短的“不同释义”提示，并允许查看两种来源原文；
- 不得通过改写底稿、删去来源或生成折中解释来掩盖差异。

每个原文单元都必须记录“未比较、无差异、有差异、不适用”之一。讲稿有对应段落时，正式课程只能使用“无差异”或“有差异”；讲稿无对应段落时使用“不适用”并记录原因。“未比较”一律阻断课程确认。以下任一情况视为会影响理解并需要在课程中展示：关键字词含义不同、对原句主旨的判断不同、或对学习者应采取的行为给出不同结论。最终判断由产品所有者确认并留痕。

### 2.3 课程划分原则

主学习路径不再以“连续21天”为承诺，也不要求课程总数恰好为21课。课程应按以下原则重新划分：

1. 以《弟子规》的篇章顺序和完整语义为主线；
2. 结合相关讲稿内容密度，确保一课能够形成相对完整的理解主题；
3. 以约20分钟完成五步主流程为负荷目标；
4. 目标范围为18—24课；超出范围时必须记录原因并由产品所有者批准书面例外，批准后仍可通过验收；
5. 不得为凑课程数量拆散完整语义，也不得把过多原文或讲稿塞入同一课；
6. 课程标题和导语只能中性描述学习范围，不新增价值判断；
7. 原文没有直接对应讲稿时，课程仍然保留，使用原文与公开底稿完成基础学习，并明确标记“暂未找到直接对应讲稿”。

每课讲稿必读重点目标为1200—2200个汉字，统计时排除空白和标点，以正常阅读约5—8分钟为目标。超出范围时必须记录原因并由产品所有者批准例外。每课在内容审核时进行一次完整流程试读，15—25分钟视为符合“约20分钟”目标；超出时同样需要书面例外。与本课有关的完整讲次仍可从课程进入，但不计入20分钟必修流程。

### 2.4 课程与讲稿映射模型

课程与40讲采用段落级、多对多映射，不使用“第5—8讲全部相关”一类无法精确定位的范围映射。

每一课至少记录：

- 课程 ID、篇章、标题、顺序和预计时长；
- 本课包含的原文语义单元 ID；
- 对应的底稿注释与译文片段；
- 直接相关的讲次与段落 ID；
- 纳入五步主流程的讲稿重点段落；
- 仅供深入阅读的相关讲次；
- 未找到讲稿对应内容的原文单元；
- 讲稿与底稿的差异记录；
- 反思问题及其来源；
- 内容审核状态与确认记录。

段落映射可以由 AI 根据原文、主题和讲稿正文生成候选结果，但 AI 结果不得直接成为正式课程数据。每一个映射都必须能够打开并核对原讲稿段落，最终由产品所有者人工确认。

讲稿映射锚定不可修改的原始正文段落 ID。展示版需要拆分或合并段落时，使用独立的展示片段 ID，并保存展示片段到一个或多个原始段落的映射；课程映射、搜索定位和阅读位置不得因展示分段变化而失效。

### 2.5 内容编排与人工审核流程

内容编排在小程序外部的 Markdown 或 JSON 文件中完成。小程序只读取已经确认的数据，不提供课程编辑后台。

每课依次经过以下状态：

1. **草稿**：完成初步原文分组，并生成底稿和讲稿候选映射；
2. **待审核**：课程范围、讲稿重点、来源和反思问题已经齐备，等待产品所有者核对；
3. **已确认**：产品所有者已逐项确认，可以进入正式小程序数据。

单课审核清单必须覆盖：

- 原文字句、顺序和语义分组正确；
- 生僻字读音经过核对；
- 底稿注释和译文引用位置正确；
- 讲稿重点确实与本课原文直接相关；
- 所有讲稿内容可以定位到具体讲次和段落；
- 必读讲稿阅读量符合范围；
- 内容冲突已经记录并按规则处理；
- 所有原文单元的差异比较状态均为“无差异、有差异、不适用”之一，不存在“未比较”；
- 反思问题有明确讲稿来源，或属于允许使用的中性通用问题；
- 不存在产品自行扩写的现代解释；
- 课程状态由产品所有者明确改为“已确认”。

没有适合讲稿来源时，只允许使用以下固定通用问题，不得临时生成其他无来源问题：“这一课中，哪一句或哪段讲解对你最有启发？为什么？”

任何未确认课程均可用于内容预览，但不得被打包为正式学习课程。

### 2.6 内容验收标准

内容数据必须同时通过人工审核和自动检查。

自动检查至少覆盖：

- 来源 ID、原文单元 ID、讲次 ID 和段落 ID 均存在且唯一；
- 课程引用不存在失效段落；
- 原文没有意外遗漏或重复归入主课程；
- 讲稿重点不存在无来源正文；
- 单课必读讲稿字数和试读时长在约定范围内，超出时必须存在已批准的例外记录；
- 同一讲稿段落被重复设为多个课程重点时给出提示，但允许人工确认合理的多课复用；
- 未找到讲稿对应内容的原文已经显式记录；
- 存在内容差异时已经提供差异记录；
- 正式课程不存在差异比较状态为“未比较”的原文单元；
- 反思问题满足来源规则；
- 所有正式课程均为“已确认”状态；
- 所有告警已经消除，或关联到产品所有者已批准的例外记录；被批准的告警在检查报告中标记为“例外通过”，计入检查通过结果但不得隐藏。

本部分的完成标准是：课程划分结果已经确定；每课均具备完整来源链和审核记录；全部正式课程通过自动检查；产品所有者能够从课程中的任意解释或讲稿段落返回原始来源。

## 3. 学习体验与功能需求

### 3.1 信息架构

产品使用五个一级入口：

1. **学习**：继续上次课程、开始建议课程、处理到期复习；
2. **课程**：按《弟子规》篇章浏览完整系统课程；
3. **讲稿**：按讲次浏览蔡老师《细讲弟子规》40讲全文；
4. **复习**：处理到期背诵内容，并查看全部评级记录；
5. **我的**：查看学习事实、反思、资料来源、备份恢复与本地设置。

内容搜索从“课程”和“讲稿”页面顶部进入，不单独占用一级入口。原 v1.1.0 的“档案”并入“我的”，“AI助手”入口删除。

所有页面应保持安静、清晰、适合长文字阅读的视觉气质，但不要求兼容 v1.1.0 的页面结构或布局。

### 3.2 学习首页

学习首页按以下优先级组织内容：

1. **到期复习**：显示今天到期的背诵条目数量和“开始复习”操作；
2. **继续学习**：存在未完成课程时，显示课程名称、当前步骤和继续入口；
3. **下一课建议**：不存在未完成课程时，显示按课程顺序推荐的下一课；
4. **最近位置**：提供最近打开的课程或讲稿入口；
5. **总体记录**：显示已完成课程数和课程总数。

到期复习只作为优先建议，不阻止用户直接进入新课。首页不得显示连续学习天数、排名、能力分数或完成焦虑类提示。

同一时间只允许一个活动学习轮次。用户从另一课程开始正式学习时，现有轮次自动保存为暂停状态，新轮次成为首页“继续学习”的对象；暂停轮次仍可从课程列表继续，最近操作的活动或暂停轮次优先显示。

首次使用且没有学习记录时，首页直接介绍系统课程结构并提供“开始第一课”。全部课程完成后，首页继续提供到期复习、讲稿阅读和重新学习入口，不生成毕业证书或综合能力评价。

### 3.3 系统课程

课程页按《弟子规》篇章顺序展示约18—24课。每课卡片至少显示课程顺序、标题、包含的原文范围、预计用时和实际状态。

课程状态包括：

- 未开始；
- 学习中，并显示当前步骤；
- 已完成。

全部课程始终可以打开。界面通过顺序、状态和“建议下一课”提供路径引导，不使用强制锁定。自由查看未开始或已完成课程不会自动改变学习状态。

已完成课程再次打开时默认进入查阅模式，并保留原完成记录。用户只有主动选择“重新学习本课”后，才创建新的学习轮次；课程在列表中仍保留“已完成”事实，并附加显示“重新学习中”。同一课程存在未完成重学轮次时，再次选择重新学习必须继续该轮次，不重复创建。重学轮次可以暂停，但不得删除历史完成事实、反思或背诵评级；完成后记录该轮次的完成时间。

课程详情必须显示本课原文范围、五步学习入口、进度、相关讲次和来源信息。讲稿未直接覆盖的原文必须按内容规则显示对应说明。

### 3.4 五步学习流程

每课固定采用五步主流程。五个步骤始终可以自由前后切换，不强制逐步解锁。

1. **复习旧课**
   - 第一次进入本轮次“复习旧课”时生成固定快照：候选原文单元必须在快照生成前已经存在至少一条评级历史、到期日不晚于当前本地日期、且不属于当前课程；自由练习产生的评级历史同样计入。候选依次按到期日升序、最后评级时间升序、规范原文顺序升序、原文单元 ID 升序排列，抽取前3个；
   - 快照生成后，本步期间新到期的内容不自动加入；完成快照内全部条目，或明确选择“本次跳过”，即可完成本步；
   - 没有到期内容时显示“当前没有到期复习”，用户仍可明确完成本步；
   - 复习结果进入统一背诵评级和排期规则。
2. **学习原文**
   - 展示本课《弟子规》原文、生僻字读音、公开底稿中的注释与基础译文；
   - 明确标注内容来源；
   - 不提供全文拼音或音频朗读。
3. **课程讲解**
   - 展示经人工确认的讲稿重点段落及讲次来源；
   - 必读重点目标阅读量为1200—2200字；
   - 提供进入相关完整讲次的入口；
   - 存在重要释义差异时，按第2.2节规则提供查看入口。
4. **背诵练习**
   - 依次提供显示全文、关键词提示、隐藏全文、显示全文对照和自我评级；
   - 不使用键盘默写、语音识别或自动相似度评分；
   - 评级以原文单元为对象，不允许用一个评级批量覆盖整课；本课每个原文单元都完成一次口头背诵尝试并选择评级后，才可标记本步完成。
5. **理解反思**
   - 每课只显示一个问题；
   - 问题优先来自讲稿中的提问、案例或核心观点，并标明讲次；没有合适来源时使用中性通用问题；
   - 每个学习轮次创建独立反思记录；用户可以写下反思，也可以仅标记“已思考”；
   - 不评分、不限制字数。

每一步都通过明确的“完成本步”操作改变状态。该操作表示用户确认已完成学习动作，不是系统对阅读质量的客观证明；滚动到底、停留时长或打开页面不得自动视为完成。五步全部完成后，本学习轮次标记为已完成。

系统必须随输入和操作自动保存当前步骤、步骤完成状态、背诵评级、反思内容和相关阅读位置。中途退出后应从保存位置继续，不要求用户重新完成已经保存的步骤。

### 3.5 背诵与间隔复习

背诵内容以原文语义单元为最小复习对象。每次背诵或复习后，用户必须选择一个自我评级：

- **熟练**：7天后再次到期；
- **模糊**：3天后再次到期；
- **待复习**：次日再次到期。

新评级立即替代上一轮排期，并记录评级时间和下一次到期日。系统不得把评级解释为客观能力测量；完整评级历史仍然保留。

“到期”包含到期日等于或早于当前本地日期的全部条目；逾期条目按到期日从早到晚排列。“即将到期”指当前日期之后7个本地日历日内到期的条目。

复习页分为：

1. 今天到期；
2. 即将到期；
3. 全部背诵内容，可按篇章和评级筛选。

单次复习沿用“关键词提示—口头背诵—对照原文—重新评级”的流程。没有到期内容时显示清晰空状态，并允许自由选择其他原文练习。自由练习产生的新评级同样更新排期。

### 3.6 完整讲稿资料库

讲稿页完整展示蔡老师《细讲弟子规》40讲，按讲次顺序浏览。每讲至少显示讲次、标题、阅读状态和最近阅读位置。阅读状态固定为“未读、阅读中、已读”：首次打开正文后变为“阅读中”，“已读”只能由用户主动标记，也可以主动取消为“阅读中”。

讲稿正文必须：

- 使用适合长文阅读的字号、行距、段间距和内容宽度；
- 支持讲次内关键词查找和全局内容搜索定位；
- 能从段落返回引用该段的系统课程；
- 明确区分未经修改的原始正文与存在校订记录的展示版本；
- 自动保存最后阅读段落或位置。

“已读”由用户主动标记。滚动到底、停留时长或从课程打开讲稿均不得自动标记已读。课程只要求阅读已选定的讲稿重点，不要求读完整讲次。

### 3.7 内容搜索

首版提供完全离线的简体中文关键词搜索，搜索范围包括：

- 《弟子规》原文；
- 公开底稿的注释和译文；
- 课程标题与导语；
- 蔡老师40讲正文。

搜索忽略常见中文标点和空格。输入包含多个以空格分隔的关键词时，结果必须同时包含全部关键词；结果按命中次数排序，同次数保持内容原顺序。首版不支持繁简转换、拼音搜索、同义词、错别字纠正、语义搜索、自动摘要或问答。

搜索结果粒度固定如下：

- “原文”组每个原文单元最多一条结果，检索文本为该单元原文及对应底稿注释、译文；同一单元内多处命中合并；
- “课程”组每门课程最多一条结果，检索文本为课程标题和导语；
- “讲稿”组每个展示片段最多一条结果，检索文本为展示版正文；
- 一条结果的命中次数等于该结果全部检索文本中所有输入关键词的精确出现次数总和；所有关键词都至少出现一次后才进入结果；
- 同组内先按命中次数降序，再按原文顺序、课程顺序，或“讲次编号＋展示片段顺序”升序；组间固定按“原文、课程、讲稿”展示；
- 性能验收中的“前20条”指按上述组间和组内顺序得到的前20条总结果，不足20条时使用全部结果。

结果按“原文、课程、讲稿”分组。底稿注释和译文命中归入“原文”，并定位到该原文单元所属的唯一主课程；讲稿搜索使用展示版正文，原始正文只在来源查看中提供。结果必须显示命中词附近的正文片段并突出关键词。点击结果后直接定位到对应课程、讲次或段落，不只打开页面顶部。

原文结果定位到主课程中的对应原文单元，课程结果定位到课程详情，讲稿结果定位到展示片段并通过其映射保持原始段落锚点稳定。

系统必须保存最近搜索词，并提供单条删除和全部清除。没有结果时说明实际搜索范围，并建议缩短关键词；不得生成不存在的相近答案。

### 3.8 个人学习记录

“我的”页面只展示可验证的学习事实：

- 已完成课程数量、课程总数和最近学习时间；
- 已手动标记为已读的讲稿；
- 当前到期复习数量；
- 背诵内容在“熟练、模糊、待复习”中的数量；
- 按篇章展示的实际课程进度；
- 反思记录及其所属课程和更新时间；
- 内容来源说明；
- 数据备份、恢复和重置入口；
- 小、中、大三级正文字号设置。

反思记录按学习轮次保存，可以查看、修改和逐条删除。修改后更新修改时间；删除前必须明确确认。重新学习产生新反思记录，不得覆盖既有反思。

页面不得生成综合能力百分比、排名、连续打卡、毕业证书或基于完成数量推测出的理解能力。

所有主要功能都必须处理以下状态：首次使用无数据、当前无到期复习、搜索无结果、内容引用失效、数据保存失败和恢复失败。错误提示必须说明发生了什么、哪些数据受到影响以及用户可以采取的恢复操作，不得只显示“操作失败”。

## 4. 数据与技术边界

### 4.1 离线运行约束

v2.0 是仅供产品所有者个人使用的手机端微信小程序。除开发工具预览、安装或版本更新外，所有学习功能必须完全离线运行。

离线范围包括：

- 系统课程与五步学习；
- 《弟子规》原文、注释和译文阅读；
- 蔡老师40讲全文阅读；
- 背诵、自我评级与间隔复习；
- 反思记录；
- 全文搜索与段落定位；
- 学习记录、备份和恢复。

首版不建设接口、账号、云同步、远程内容服务、在线搜索、使用分析或遥测上报。不得收集或发送设备标识、学习内容、反思文本或操作记录。

允许使用微信小程序分包等本地资源组织方式控制主包体积，但所有课程与讲稿必须随可安装的小程序资源提供。不得把讲稿改为联网后才能阅读，也不得因离线状态隐藏核心内容。

课程重编开始前必须完成一次包体与真机离线容量验证，证明规范原文、公开底稿、40讲、课程数据和搜索索引能够随小程序资源完整提供；验证结果和所采用的资源组织方式必须留档。

### 4.2 内容数据模型

内容数据必须具有独立版本号，并以稳定 ID 建立关系。正文修改、标点调整或页面重排不得改变既有 ID。

核心内容实体如下：

| 实体 | 必备信息 | 关键关系 |
| --- | --- | --- |
| 来源 | 来源 ID、名称、类型、输入文件、版本状态、用途、备注 | 被原文、底稿和讲稿引用 |
| 原文单元 | 原文单元 ID、篇章、顺序、原文、生僻字读音 | 归入课程并生成背诵记录 |
| 底稿片段 | 片段 ID、对应原文单元、题解、注释、译文、来源 ID | 补充原文学习内容 |
| 讲次 | 讲次 ID、编号、标题、原始正文版本、展示正文版本、来源 ID | 包含多个讲稿段落 |
| 讲稿段落 | 段落 ID、讲次 ID、顺序、正文、校订记录 | 映射到一个或多个课程 |
| 展示片段 | 展示片段 ID、展示正文、顺序、原始段落 ID 集合 | 支持校订分段并保持来源锚点稳定 |
| 课程 | 课程 ID、篇章、顺序、标题、导语、预计时长、审核状态 | 聚合原文、底稿、讲稿重点和反思问题 |
| 映射 | 映射 ID、课程 ID、原文单元 ID、讲稿段落 ID、用途、审核记录 | 表达段落级多对多关系 |
| 反思问题 | 问题 ID、课程 ID、正文、来源类型；讲稿来源时必须有讲次或段落 ID，固定通用问题时该字段为空 | 用于五步流程的理解反思 |
| 内容差异 | 差异 ID、原文单元 ID、比较状态、底稿片段 ID、说明、展示标记、确认记录；“有差异”时讲稿段落 ID 必填，“不适用”时为空 | 记录不同释义 |

讲稿原始正文、讲稿展示版和公开学习底稿必须分开保存。展示版的每项校订至少记录受影响段落、修改前内容、修改后内容、修改原因和确认状态。

内容更新涉及删除、合并或替换 ID 时，必须随更新提供显式迁移规则。无法迁移的个人记录不得被静默丢弃。

阅读位置优先恢复到稳定内容 ID 和段内字符偏移；内容升级后该位置失效时，降级定位到所属原文单元或讲次顶部，并明确提示定位已调整。

### 4.3 学习状态模型

学习状态具有独立版本号，至少记录：

- 每课的未开始、学习中或已完成状态；
- 每个学习轮次的轮次 ID、课程 ID、类型、状态、五步完成状态、当前步骤、开始时间、最近访问时间和完成时间；
- 每个轮次“复习旧课”的快照生成状态、生成时间、候选原文单元顺序、逐项完成或评级状态，以及整步跳过状态；快照一经生成不得因退出、切课或恢复而重算；
- 每个轮次背诵步骤要求评级的原文单元集合，以及本轮本步骤已经完成评级的原文单元集合；
- 当前活动轮次及暂停轮次；
- 每个原文单元的完整评级历史；每条评级事件至少记录事件 ID、原文单元 ID、评级、评级时间、本地评级日期、来源类型（课程背诵、旧课复习、自由练习）、关联轮次 ID 和步骤（不适用时为空）；当前评级和下次到期日由最后一次事件派生；
- 每条反思的反思 ID、轮次 ID、课程 ID、正文或“已思考”状态、创建时间和修改时间；
- 每讲的“未读、阅读中、已读”状态；
- 课程和讲稿的最近阅读位置，统一使用稳定内容 ID 加段内字符偏移；
- 最近搜索词；
- 必要的本地显示设置。

所有改变学习状态的操作必须及时自动保存。保存过程不得先删除上一份有效状态；单次写入失败时，应继续保留最近一次有效数据并向用户说明哪些新操作尚未保存。

复习到期日以 `YYYY-MM-DD` 本地日历日期保存，不精确到时分秒：待复习为评级本地日期加1天、模糊加3天、熟练加7天。同一原文单元在同一天内多次评级时，以最后一次评级和时间为当前状态，但完整评级历史仍保留。设备时区或日期变化后，以设备当前本地日期重新判断是否到期，不改写已经保存的到期日。

自由查看内容不得改变课程完成状态。重新学习课程创建独立学习轮次，课程仍保留既有完成事实；新轮次不得删除历史反思或背诵评级。

### 4.4 本机存储与备份恢复

学习数据默认只保存在本机。系统必须提供手动导出和恢复，并明确提示清除小程序数据或更换设备可能造成记录丢失。

备份文件必须：

- 使用可校验的结构化格式；
- 记录备份格式版本、学习状态版本、内容数据版本和导出时间；
- 包含全部个人学习状态：课程完成事实、所有学习轮次及其旧课快照与背诵步骤完成集合、步骤状态、完整背诵评级事件、反思、讲稿阅读状态、阅读位置、最近搜索词、当前字号和其他本地显示设置；
- 不重复包含《弟子规》原文、公开底稿、课程正文或40讲全文；
- 不包含设备标识或与个人学习无关的信息。

三个版本号均使用“主版本.次版本”：

| 备份中的版本相对当前程序 | 备份格式 | 学习状态 |
| --- | --- | --- |
| 完全相同 | 直接校验 | 直接校验 |
| 同主版本、旧次版本 | 有连续迁移链则迁移，否则拒绝 | 有连续迁移链则迁移，否则拒绝 |
| 旧主版本 | 有覆盖全部主版本变化的连续迁移链则迁移，否则拒绝 | 有覆盖全部主版本变化的连续迁移链则迁移，否则拒绝 |
| 同主版本、新次版本 | 拒绝，要求使用不低于该次版本的程序 | 拒绝，要求使用不低于该次版本的程序 |
| 新主版本 | 拒绝，要求升级程序 | 拒绝，要求升级程序 |

- 迁移只允许从旧版本向当前版本执行，不提供向下迁移；
- 内容版本不要求完全相同，但备份中的内容引用必须按以下规则处理：
  - **硬引用**：课程完成事实、学习轮次和反思的课程 ID，评级事件、当前排期、旧课快照和背诵完成集合的原文单元 ID，以及讲稿阅读状态的讲次 ID。硬引用必须能直接解析或经 ID 迁移表解析，任一失败则拒绝整包导入；
  - **可降级引用**：课程或讲稿阅读位置中的展示片段 ID 和段内偏移。展示片段失效但所属课程、原文单元或讲次硬引用仍可解析时，降级到对应原文单元或讲次顶部并记录调整；
  - 最近搜索词和显示设置不含内容引用，按备份值恢复；
  - 备份导入不隔离或丢弃单条记录；除上述阅读位置降级外，无法完整恢复时必须拒绝整包导入；
- 所有迁移都在写入前于内存中完成并重新校验。迁移或校验失败时不得创建部分恢复结果，也不得修改当前状态。

恢复流程必须依次执行：

1. 读取但不立即写入备份；
2. 校验文件结构、版本和关键引用；
3. 展示待恢复数据摘要；备份格式或必要内容引用不兼容时必须中止，不允许部分导入；
4. 提醒恢复将完整覆盖当前学习状态；
5. 用户确认后，先自动保存当前状态快照；
6. 写入并重新读取恢复结果，确认数据有效；
7. 失败时自动回滚到导入前快照；
8. 成功后展示恢复的课程、背诵和反思数量。

导入前快照只保留最近一份，并在下一次成功导入前可由用户手动恢复；创建快照失败时不得开始覆盖。恢复写入、结果校验或自动回滚任一步失败时，必须停止后续操作、保留仍可读取的数据并允许导出诊断备份。

首版不提供自动合并两份学习记录。重置全部数据必须二次确认，并明确列出会删除的内容。重置不得删除内置课程和讲稿。

### 4.5 搜索、性能与异常处理

全文搜索必须基于本地内容完成。在用于真机验收的手机上，使用附录 D 的10个固定查询；在应用已启动且本地索引已可用的状态下，每个查询执行一次，从提交搜索到显示前20条或全部结果（不足20条时）完成渲染为止，10个查询中至少9个不超过500毫秒。结果点击后必须定位到实际命中段落；长讲稿不得只打开讲次顶部。

内容组织和索引方式由实现选择，但必须满足：

- 搜索结果可追溯到原始内容；
- 索引与当前内容版本一致；
- 内容更新后不会继续返回已失效段落；
- 搜索失败不影响直接浏览课程与讲稿。

系统启动和内容更新时必须检查数据版本与引用完整性。以下隔离规则用于本机既有状态或内容升级；备份导入按第4.4节执行完整校验，不适用部分隔离。发生数据损坏、版本不兼容或内容引用失效时：

1. 不静默删除个人记录；
2. 将无法识别的记录隔离，保持其原始内容可导出；
3. 继续加载能够安全识别的内容；
4. 明确说明问题类型、受影响数据和建议操作；
5. 允许导出包含问题记录的诊断备份。

关键异常至少包括：本地保存失败、存储空间不足、备份格式错误、备份版本不兼容、恢复校验失败、内容 ID 失效、搜索索引失效和离线资源缺失。每种异常都必须提供可执行的恢复路径，不得仅提示“操作失败”。

### 5.3 功能验收

功能至少在微信开发者工具和一台真实手机上完成验收。开发者工具用于构建、数据和异常测试，真机用于阅读、导航、保存、搜索和恢复体验确认。

核心验收场景包括：

| 场景 | 通过条件 |
| --- | --- |
| 首次使用 | 无学习记录时能够开始第一课，所有页面空状态正确 |
| 完整学习一课 | 五步可自由切换；状态自动保存；完成五步后课程正确完成 |
| 多原文背诵 | 一课包含多个原文单元时逐单元评级，未完成任一单元前背诵步骤不能完成 |
| 中途退出 | 退出后重新进入，恢复到最近步骤、内容和阅读位置 |
| 切换学习课程 | 开始另一课程后原轮次安全暂停，首页继续最近轮次，暂停轮次可恢复 |
| 自由浏览 | 打开任意未开始或已完成课程，不会自动改变进度 |
| 重新学习 | 主动重新学习后创建独立轮次；重复操作继续未完成轮次；历史反思和完整评级历史仍保留 |
| 背诵自评 | 三种评级均生成正确到期日，同日最后一次评级生效 |
| 到期复习 | 到期及逾期内容出现在首页和复习页；使用同日到期超过3项的测试数据验证旧课快照稳定排序、持久化和恢复；重新评级后排期更新 |
| 讲稿阅读 | 40讲完整可打开，阅读位置可恢复，已读状态只能手动改变 |
| 内容搜索 | 原文、底稿、课程和讲稿均可命中，结果能定位到实际段落 |
| 反思记录 | 可以创建、自动保存、修改和确认删除 |
| 字号设置 | 小、中、大三级字号在长文页面生效并在重启后保留 |
| 备份导出 | 备份包含规定的学习状态，不包含课程和讲稿正文 |
| 备份恢复 | 导入前显示摘要并创建快照，覆盖后全部个人状态与备份一致，快照可以手动恢复 |
| 跨版本恢复 | 分别覆盖版本相同、旧次、旧主、新次、新主五类备份格式和学习状态；只有具备完整迁移链的旧版本可迁移恢复，新版本、缺少迁移链或存在无法解析必要引用时安全拒绝且不修改当前状态 |
| 跨内容版本恢复 | 硬引用可解析或迁移时允许恢复；展示片段阅读位置失效时按规则降级；任一硬引用无法解析时整包拒绝且不修改当前状态 |
| 恢复失败 | 格式错误或不兼容时不破坏现有状态，并给出可执行说明 |
| 完全离线 | 断网后课程、讲稿、复习、搜索和记录均正常工作 |
| 数据异常 | 引用失效或状态损坏时不静默删除记录，允许导出诊断数据 |

复习日期测试使用可控测试数据覆盖次日、3天、7天和同日重复评级，不要求验收人员真实等待对应天数。

搜索性能使用第4.5节定义的10个固定查询验收，并保存每次耗时。真机验收记录必须注明设备型号、系统版本、微信版本、小程序内容版本、测试日期和结果。

以下任一问题均阻断验收：构建失败、运行时报错、正式内容引用失效、学习状态丢失、备份无法恢复、搜索不能定位、离线核心内容缺失或未确认课程进入正式数据。

### 5.4 内容验收

全部正式课程必须通过第2.5节的人工审核和第2.6节的自动检查。内容验收不得抽样替代逐课确认。

通过条件包括：

- 规范原文文件已由产品所有者逐句确认并登记校验值；全部原文完整覆盖，无意外遗漏、重复或顺序错误；
- 最终课程数量和每课负荷符合划分原则，或具有产品所有者批准的例外记录；
- 受控讲稿快照中的第1—40讲全部进入产品，无讲次缺失或相对于该快照的正文截断；由于缺少外部原始版本，本验收不声称快照与某一出版或网络版本完全一致；
- 每课讲稿重点均能定位到具体讲次和段落；
- 讲稿重点字数和完整流程试读时长符合目标范围，或具有产品所有者批准的例外记录；
- 未找到讲稿对应内容的原文已经明确标记；
- 讲稿与底稿差异已经记录并按规则展示；
- 所有正式课程原文单元的差异比较状态均为“无差异、有差异、不适用”，不存在“未比较”；
- 所有生僻字读音已经核对；
- 所有反思问题具有讲稿来源或符合中性通用问题规则；
- 不存在产品自行扩写的现代解释；
- 所有正式课程状态均为“已确认”；
- 所有来源、原文、课程、讲次、段落和映射 ID 通过完整性检查。

内容自动检查必须100%通过。“通过”包括无告警项目和关联到已批准例外、被标记为“例外通过”的项目；未处理告警不得通过。不得通过关闭检查或删除告警规避问题。
